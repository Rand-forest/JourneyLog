# JourneyLog
JourneyLog Travel Itinerary and Expense Tracker

🌐 How to Put Your JourneyLog App on the Internet (For Free!)

Welcome! You have a completed piece of code for your Travel Itinerary App, and now you want to make it a real website that you can visit on your phone or share with others.

We are going to use a free service called GitHub Pages. Think of GitHub as a giant digital filing cabinet where people store website files, and "GitHub Pages" as the magic button that turns those files into a live website.

This guide is completely dummy-proof. Just follow the steps one by one!

**Step 1: Save your Website File to your Computer**
---------------------------------------------------

Right now, your website code is just text on your screen. We need to save it as a file.

Open a simple text editor on your computer (like Notepad on Windows or TextEdit on Mac).

Copy all the code for your index.html file (start from <!DOCTYPE html> all the way down to </html>).

Paste that code into your text editor.

Click File > Save As...

Name the file exactly this: index.html

Important: Make sure it doesn't save as index.html.txt. It must end in .html.

Save it somewhere easy to find, like your Desktop.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

**Step 2: Create a Free GitHub Account**
----------------------------------------

If you already have a GitHub account, you can skip this step!

Go to https://github.com

Click the Sign up button in the top right corner.

Follow the prompts to enter your email, create a password, and pick a username.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

**Step 3: Create a "Repository" (A Project Folder)**
----------------------------------------------------

A "repository" is just GitHub's fancy word for a folder that holds your project.

Once you are logged into GitHub, look at the very top right of the screen.

Click the small + icon next to your profile picture.

Click New repository from the dropdown menu.

Repository name: Type a name for your project (for example: my-travel-app). No spaces allowed, use dashes instead!

Keep it set to Public (this is required for the free website hosting).

Scroll all the way down to the bottom and click the green Create repository button.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

**Step 4: Upload your Website File**
------------------------------------

Now we put your index.html file into your new project folder.

On the screen that appears after creating your repository, look for the small blue text that says uploading an existing file (it's near the middle of the screen) and click it.

A drag-and-drop screen will appear.

Drag the index.html file from your Desktop (or wherever you saved it) and drop it into that box.

Wait a few seconds for it to load.

Scroll down and click the green Commit changes button. (Note: "Commit" is just computer-speak for "Save").

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

**Step 5: Turn on the Magic (Enable GitHub Pages)**
---------------------------------------------------

Your file is safely stored! Now we tell GitHub to display it as a website.

Look near the top of your repository page for a row of tabs (Code, Issues, Pull requests, etc.). Click the Settings tab (it has a little gear icon ⚙️).

On the left-hand menu of the Settings page, look down the list and click Pages.

Under the "Build and deployment" section, look for the dropdown menu labeled Branch.

Click the dropdown that currently says None and change it to main.

Leave the next dropdown as / (root).

Click the Save button right next to it.

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

**Step 6: Get Your Live Link! 🎉**
-----------------------------------

You are done! But you have to wait just a minute.

GitHub takes about 1 to 2 minutes to build your website for the first time. Go grab a sip of water!

After a minute or two, refresh the Pages screen.

At the very top of the screen, a message will appear saying: "Your site is live at [Your Web Link]".

Click that link! It will open your fully functioning JourneyLog app in a new tab.

Bookmark that link! You can now visit it on your phone, tablet, or send it to your travel buddies. Because you set up the Google Sheet backend, everything you type into this website will automatically save to your Google Sheet!

------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

**Need to make an update later?**
---------------------------------

If you ever want to change the code (like changing the title or colors):

Go back to your repository on GitHub.

Click on the index.html file.

Click the little Pencil icon ✏️ in the top right corner of the code box to edit it.

Paste your new code.

Click the green Commit changes... button at the top right.

Wait 2 minutes, and your live website will automatically update!
